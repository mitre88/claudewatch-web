Resolución de Problemas

Sigue estos pasos según el problema que experimentes.

🔍 El servidor no aparece en la app

  • Verifica que tu iPhone y tu Mac estén en la misma red WiFi.
  • En tu Mac ejecuta ~/Library/ClaudeWatch/claudewatch link. Ese comando levanta el servidor si hace falta y reabre la página de pairing.
  • Si ese comando no existe, vuelve a correr la instalación con curl -fsSL https://claudewatch-web.vercel.app/install.sh | bash. Pégalo completo y no uses sudo.
  • Revisa que el firewall de tu Mac no esté bloqueando el puerto 7432 (Preferencias del Sistema > Seguridad y Privacidad > Firewall).
  • Reinicia la búsqueda en la app arrastrando hacia abajo en la lista de dispositivos.

📡 La conexión se pierde frecuentemente

  • Asegúrate de que tu Mac no entre en modo de suspensión mientras usas la app (Ajustes del Sistema > Batería > Evitar suspensión).
  • Verifica que la señal WiFi sea estable en ambos dispositivos.
  • Si usas una red con bandas 2.4GHz y 5GHz separadas, conecta ambos dispositivos a la misma banda.
  • Reinicia el servidor con ~/Library/ClaudeWatch/claudewatch restart en Terminal.

📷 El código QR no se escanea

  • Verifica que hayas concedido permisos de cámara a ClaudeWatch en Ajustes > ClaudeWatch > Cámara.
  • Asegúrate de que haya buena iluminación y que el QR esté completamente visible en el visor.
  • Intenta limpiar la pantalla de tu Mac y alejar ligeramente el iPhone.
  • Como alternativa, usa la opción "Ingresar IP manualmente" en la pantalla de emparejamiento.
  • Si necesitas un QR nuevo, vuelve a abrir pairing con ~/Library/ClaudeWatch/claudewatch link.

🔔 No recibo notificaciones

  • Ve a Ajustes > Notificaciones > ClaudeWatch y asegúrate de que las notificaciones estén habilitadas.
  • Verifica que No Molestar o Enfoque no estén silenciando las notificaciones de ClaudeWatch.
  • En la app, revisa Ajustes > Notificaciones y asegúrate de que los tipos de alerta deseados estén activados.
  • Fuerza el cierre de la app y vuelve a abrirla para restablecer la conexión WebSocket.

⚡ El servidor no inicia al encender el Mac

  • Ejecuta claudewatch install-launchagent para instalar el inicio automático.
  • Verifica que el agente esté registrado: launchctl list | grep claudewatch.
  • Si ves un código de error diferente a 0, revisa los logs en ~/Library/Logs/ClaudeWatch/.

🔄 La app muestra datos desactualizados

  • Arrastra hacia abajo en la pantalla principal para forzar una actualización.
  • Verifica el indicador de estado en la esquina superior derecha — un punto rojo indica desconexión.
  • Si el indicador es verde pero los datos no se actualizan, toca el ícono de reconexión.
  • Reinicia tanto la app como el servidor y vuelve a conectar.

Troubleshooting

Follow these steps based on the issue you're experiencing.

🔍 Server not appearing in app

  • Verify your iPhone and Mac are on the same WiFi network.
  • On your Mac run ~/Library/ClaudeWatch/claudewatch link. That command starts the server if needed and reopens the pairing page.
  • If that command is missing, rerun the installer with curl -fsSL https://claudewatch-web.vercel.app/install.sh | bash. Paste it exactly and do not use sudo.
  • Check that your Mac's firewall isn't blocking port 7432 (System Preferences > Security & Privacy > Firewall).
  • Restart the search in the app by pulling down on the device list.

📡 Connection drops frequently

  • Make sure your Mac doesn't sleep while using the app (System Settings > Battery > Prevent sleep).
  • Verify the WiFi signal is stable on both devices.
  • If using a network with separate 2.4GHz and 5GHz bands, connect both devices to the same band.
  • Restart the server with ~/Library/ClaudeWatch/claudewatch restart in Terminal.

📷 QR code won't scan

  • Check that you've granted camera permission to ClaudeWatch in Settings > ClaudeWatch > Camera.
  • Make sure there is good lighting and the QR is fully visible in the viewfinder.
  • Try cleaning your Mac screen and holding the iPhone slightly further away.
  • As an alternative, use the "Enter IP manually" option on the pairing screen.
  • If you need a fresh QR, reopen pairing with ~/Library/ClaudeWatch/claudewatch link.

🔔 Not receiving notifications

  • Go to Settings > Notifications > ClaudeWatch and make sure notifications are enabled.
  • Check that Focus or Do Not Disturb modes aren't silencing ClaudeWatch notifications.
  • In the app, check Settings > Notifications and make sure desired alert types are toggled on.
  • Force-quit the app and reopen it to reestablish the WebSocket connection.

⚡ Server doesn't start on Mac boot

  • Run claudewatch install-launchagent to install auto-start.
  • Verify the agent is registered: launchctl list | grep claudewatch.
  • If you see an error code other than 0, check logs at ~/Library/Logs/ClaudeWatch/.

🔄 App shows stale data

  • Pull down on the main screen to force a refresh.
  • Check the status indicator in the top right — a red dot means disconnected.
  • If the indicator is green but data isn't updating, tap the reconnect icon.
  • Restart both the app and the server and reconnect.
